# cy-417 旅游行程规划助手

请根据以下要求生成项目代码和 README。

## 项目主要需求

### 1. 项目概述

**旅游行程规划助手（TripWeaver）** 是一款纯前端旅行规划应用。用户可创建旅行计划、搜索景点、编排每日行程、管理预算与同行人，并生成可分享的行程单。所有数据持久化存储在浏览器 localStorage / IndexedDB 中。项目类型：**Web 前端应用**。

### 2. 核心数据模型（≥ 3 个，必须独立成文件）

- **Trip（旅行计划）**
  - 字段：id, title, destination, start_date, end_date, budget, currency, members（数组）, status（规划中/进行中/已结束）, created_at
  - 关键关系：一个 Trip 包含多个 DayPlan
  - 业务动作：创建旅行、编辑信息、设置预算、添加同行人
  - **此模型必须独立定义在 `src/models/trip.ts`**

- **Spot（景点/地点）**
  - 字段：id, name, category（自然风光/人文历史/美食购物/娱乐休闲）, address, lat, lng, rating, price, open_time, tags, image
  - 关键关系：被多个 DayPlan 引用
  - 业务动作：搜索景点、按分类筛选、收藏
  - **此模型必须独立定义在 `src/models/spot.ts`**

- **DayPlan（每日行程）**
  - 字段：id, trip_id, day_index, date, items（数组：spot_id, start_time, end_time, note, transport）
  - 关键关系：隶属于 Trip，引用 Spot
  - 业务动作：添加景点到某天、调整顺序、设置停留时间、交通方式
  - **此模型必须独立定义在 `src/models/dayPlan.ts`**

### 3. 核心页面（≥ 4 个，至少 2 个页面共用同一业务组件）

- **/trips（我的旅行）**
  - 消费模型：Trip
  - 关键交互：旅行计划列表、新建旅行、删除、进入详情
  - 共享组件：`<TripCard>`、`<EmptyState>`

- **/trip/:id（行程详情）**
  - 消费模型：Trip、DayPlan、Spot
  - 关键交互：查看每日行程、添加景点、调整顺序、预算统计
  - 共享组件：`<DayTimeline>`、`<SpotCard>`、`<BudgetChart>`

- **/spots（景点探索）**
  - 消费模型：Spot、Trip
  - 关键交互：搜索景点、分类筛选、收藏、添加到指定行程的某天
  - 共享组件：`<SpotCard>`（与 /trip/:id 共用）、`<CategoryFilter>`

- **/planner/:tripId/:dayIndex（行程编排）**
  - 消费模型：Trip、DayPlan、Spot
  - 关键交互：拖拽排序、设置停留时间、选择交通方式、实时预算计算
  - 共享组件：`<DayTimeline>`（与 /trip/:id 共用）、`<SpotMiniCard>`

- **/share（分享预览）**
  - 消费模型：Trip、DayPlan、Spot
  - 关键交互：生成可分享行程单、导出图片/文本
  - 共享组件：`<DayTimeline>`（与 /trip/:id 共用）、`<TripHeader>`

### 4. 横切关注点（≥ 2 个）

- **主题切换/地图主题**
  - 触达文件层：
    - `stores/themeStore.ts`
    - `constants/themes.ts`
    - 高德地图主题适配
    - 多个组件读取 theme 状态

- **全局错误处理/提示**
  - 触达文件层：
    - `utils/message.ts`
    - `components/common/GlobalErrorBoundary.tsx`
    - 所有 store 操作调用 message

- **本地数据持久化与版本兼容**
  - 触达文件层：
    - `utils/storage.ts`
    - 所有 store
    - `constants/storageVersion.ts`

### 5. 共享枚举/常量（≥ 2 组）

- **SpotCategory（景点分类）**
  - 值：NATURE = 'nature', CULTURE = 'culture', FOOD = 'food', ENTERTAINMENT = 'entertainment'
  - 定义位置：`src/constants/spot.ts`
  - 出现位置：Spot 模型、景点探索页、CategoryFilter、SpotCard、行程详情、formatters
  - **README 必须列出该枚举的所有出现位置**

- **TripStatus（旅行状态）**
  - 值：PLANNING = 'planning', ONGOING = 'ongoing', FINISHED = 'finished'
  - 定义位置：`src/constants/trip.ts`
  - 出现位置：Trip 模型、TripCard、旅行列表筛选、formatters
  - **README 必须列出该枚举的所有出现位置**

### 6. 共享前端组件清单（≥ 3 个）

- `<TripCard>`：被 /trips 使用
- `<SpotCard>`：被 /trip/:id 和 /spots 共用
- `<DayTimeline>`：被 /trip/:id、/planner、/share 共用
- `<CategoryFilter>`：被 /spots 使用
- `<SpotMiniCard>`：被 /planner 使用
- `<BudgetChart>`：被 /trip/:id 使用
- `<TripHeader>`：被 /share 使用
- `<EmptyState>`：被多个列表页共用

### 7. 共享 hooks/utils 清单（≥ 2 个）

- `useTripStats()`：旅行天数、预算、景点数量统计
- `useLocalStorage()`：localStorage 读写封装
- `useMapSpots()`：高德地图相关 spot 坐标处理
- `utils/storage.ts`：localStorage / IndexedDB 封装
- `utils/budgetCalculator.ts`：预算汇总与分摊计算
- `utils/formatters.ts`：分类文本、状态文本、时间格式化、货币格式化

### 8. 状态管理要求

必须使用 Pinia 独立 store：
```
src/stores/
├── tripStore.ts
├── spotStore.ts
├── dayPlanStore.ts
└── themeStore.ts
```

禁止全部用组件内部 state。每个 store 按实体分文件，组件中按需引入。

### 9. 技术要求

- **英文短名（项目标识）**：`tripweaver`
- **前端端口**：18417
- **前端框架**：Vue 3 + TypeScript
- **UI 库**：Element Plus + ECharts + 高德地图 API
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **数据持久化**：localStorage + IndexedDB（使用 Dexie.js）
- **其他依赖**：dayjs、lodash-es、sortablejs（拖拽排序）


**严禁合并职责到单一文件。即使是简单功能，也必须按照上述分层结构拆分，确保后续每一个小改动都至少触达 3-5 个文件。**

### 11. 特殊约束：屎山代码设计（低内聚、高耦合、牵一发动全身）

1. **常量/工具类必须多处耦合**
   - `utils/formatters.ts` 同时包含日期格式化、分类文本、状态文本、交通方式文本、货币格式化
   - `constants/messages.ts` 同时包含页面提示文案、表单校验文案、空状态文案
   - 修改一个文案可能同时影响多个页面和组件

2. **枚举必须被多处引用**
   - SpotCategory 和 TripStatus 必须在 models、constants、components、stores、formatters、router meta、筛选器中同时存在
   - 新增一个枚举值时，必须修改至少 8 处文件

3. **预算计算必须跨多处**
   - 预算规则在 `utils/budgetCalculator.ts`、DayPlan 模型、TripStore、BudgetChart、行程详情页中同时存在
   - 修改计算规则需要同步修改至少 5 处

4. **高德地图相关配置必须多处引用**
   - 地图 key、主题、marker 样式在 `config/map.ts`、SpotCard、DayTimeline、Planner 页中同时存在
   - 修改地图配置需要同步修改至少 4 处

5. **牵一发动全身的验证标准**
   - 后续如果要求"给 Spot 新增一个 'shopping' 分类"，必须触达：Spot 模型、SpotCategory 枚举、spotStore、景点探索页、CategoryFilter、SpotCard、行程详情、formatters、messages、高德地图筛选器等 ≥ 10 个文件
